1. Domain Overview
The business domain is Credit Acquisition. This involves the entire process of a customer applying for a credit product (like a credit card or loan) from the initial application start to the final decision (approval or rejection). The reports focus on monitoring application volumes, analyzing the conversion funnel, understanding applicant demographics, and evaluating the performance of different acquisition channels.

2. List of Entity Name with a description
- Application: Represents a single instance of a customer applying for a credit product. It captures the journey of the application from start to finish.
- Applicant: Represents the individual customer who is applying for credit. It holds demographic and financial information about the person.
- Credit Product: Represents the specific financial product for which the application is made (e.g., a particular credit card or loan type).
- Acquisition Channel: Represents the source or medium through which the application was submitted (e.g., online portal, mobile app, physical branch).

3. List of Attributes for each Entity with a description for each attribute

Entity: Application
- Application Started Date: The date and time when the application process was first initiated by the applicant.
- Application Submitted Date: The date and time when the applicant completed and formally submitted the application for review.
- Application Status: The current state of the application in the workflow (e.g., Started, Submitted, Approved, Rejected).
- Approved Credit Amount: For approved applications, this is the total amount of credit that has been granted.

Entity: Applicant
- Applicant Age: The age of the applicant at the time of submitting the application.
- Applicant Annual Income: The gross annual income of the applicant as declared in the application.
- Applicant State: The state where the applicant resides.
- Applicant City: The city where the applicant resides.
- Employment Type: The employment status of the applicant (e.g., Salaried, Self-Employed, Unemployed).
- Credit Score: The numerical credit score of the applicant that was used to evaluate the application.

Entity: Credit Product
- Product Name: The official name of the credit product being applied for (e.g., 'Platinum Rewards Card', '15-Year Fixed Mortgage').

Entity: Acquisition Channel
- Channel Name: The name of the channel through which the application was acquired (e.g., 'Online Portal', 'Mobile App', 'Branch', 'Partner').

4. KPI List
- Number of Applications Started: Total count of applications that were initiated.
- Number of Applications Submitted: Total count of applications that were successfully submitted for review.
- Number of Applications Approved: Total count of applications that were approved.
- Number of Applications Rejected: Total count of applications that were rejected.
- Approval Rate: The percentage of submitted applications that were approved (Calculated as: Number of Applications Approved / Number of Applications Submitted).
- Average Applicant Age: The average age of all applicants.
- Average Applicant Annual Income: The average annual income of all applicants.
- Average Credit Score: The average credit score of all applicants.
- Total Approved Credit Amount: The sum of all credit amounts granted for approved applications.

The conceptual model consists of a central 'Application' entity connected to 'Applicant', 'Credit Product', and 'Acquisition Channel' entities. The relationships are described below. (Note: Key fields are conceptual placeholders to illustrate relationships).

6. Common Data Elements in Report Requirements
The following data elements are referenced across multiple reports:
- Application Status: Used to filter and aggregate data in the funnel report and demographic analysis.
- Acquisition Channel: Used as a primary dimension for analysis in both the funnel report and the channel performance report.
- Credit Product: Used as a dimension in the funnel report and the channel performance report.
- Credit Score: Used as a metric and for analysis in the demographic report and the channel performance report.
